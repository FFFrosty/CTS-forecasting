## 二、赛题任务

### 赛题A：拖轮作业数量预测

**活跃判定：** SOG ∈ [2, 10] 且该小时内 ≥ 3条AIS记录

**空间划分：** 以 (117.79°E, 38.97°N) 为中心，3层同心圆

| 圈层 | 半径 |
|------|------|
| 核心区 | 0~3 km |
| 近港区 | 3~10 km |
| 外围区 | 10~30 km |

各圈层均值：核心区 5.5艘/h，近港区 3.2艘/h，外围区 2.1艘/h

**预测目标：** 7天×24h×3圈层 = 504个值

### 赛题B：拖轮进出流量预测

迁移判定：同一船相邻小时窗口位于不同圈层。

6个方向：核心↔近港、核心↔外围、近港↔外围，每小时流量绝大多数为 0~2 艘（最大值7）。

**预测目标：** 167个时间步×6方向 = 1002个值

### 评价指标

```
Score = SSE_TaskA + 3 × SSE_TaskB
```

## 三、EDA 关键发现

### 3.1 天气事件（核心发现）

24天数据并非平稳序列。1月13-15日出现一次明显的作业量崩塌和反弹：

```
日期        活拖轮·小时    出现船舶数
01-06(六)     222            61
01-07(日)     274            60       ← 正常作业
01-12(五)     196            57
01-13(六)     157            55       ← 崩塌开始
01-14(日)     101            53       ← 最低点
01-15(一)     113            47
01-16(二)     144            53
01-17(三)     122            48
01-18(四)     155            46
01-19(五)     196            52       ← 恢复
01-20(六)     291            55       ← 反弹超过正常
01-21(日)     284            54
```

推断：1月中旬寒潮/大风导致港口停摆约3天，积压船舶后续集中处理。日常222-275的范围被拉成了101-291。

### 3.2 船舶数 ≠ 作业量

每日出现的船舶数和每天总活跃小时数的相关系数只有 **0.40**。1月14日53艘船只产出101个小时作业，1月20日55艘船产出291个小时——船数同级，作业量差3倍。验证集给出的"每日船舶数"信息对预测作业量**帮助有限**。

### 3.3 周内模式

| 星期 | 日均活拖轮·小时 | 标准差 | CV |
|------|:-----:|:----:|:---:|
| 周日 | 220 | 103 | 47% |
| 周六 | 223 | 67 | 30% |
| 周一 | 198 | 59 | 30% |
| 周五 | 207 | **20** | **10%** |

周末波动极大（天气事件多落在周末），周五最稳定。

### 3.4 基线回测结果

滑动窗口验证（前N天训练→预测后续3天）：

| 策略 | Task A MAE | Task A RMSE | Task B MAE |
|------|:---------:|:----------:|:---------:|
| 同星期同时刻历史均值（最佳） | 2.14 | 2.91 | 0.35 |
| 最后24h均值外推 | 2.60 | 3.41 | 0.51 |
| 全历史均值常数 | 2.79 | 3.49 | 0.54 |

> 结论：周周期+日周期是最强信号。但MAE 2.14对于均值5.5的序列来说仍有较大空间（约39%误差），天气事件时的预测更是完全失效。

## 四、两条技术路线

| | 路线一：聚合统计 | 路线二：个体船建模 |
|---|---|---|
| **思路** | 直接对每小时的数字做时序预测 | 逐船预测"该小时是否活跃"→求和 |
| **优点** | 简单、过拟合风险低 | 保留每条船的出勤规律，信息利用率高 |
| **缺点** | 抹平个体差异，对天气事件敏感 | 验证期不知道哪些船会出现，需先猜 |
| **关键假设** | 历史数字模式延续 | 同星期同模式的船大概率重复出现 |

两条路可以同时跑，交叉验证比好坏。**目前队友代码走的都是路线一。**

## 五、关键待解决问题

1. **天气事件如何建模？** 训练集里有明显的崩塌-反弹模式，如果验证期也来一次，纯历史均值必死。需要引入趋势/动量信号（如近N天变化率、当前值偏离历史同期的程度）。

2. **验证期船舶数怎么用？** 用不了直接预测作业量，但可以作为一个弱约束——如果预测的某天总作业量远超历史同船数对应范围，可能不合理。

3. **B题迁移量太小怎么办？** 多数方向每小时0~2艘，预测空间极其稀疏。分类思路（预测"有没有迁移"）可能比回归（预测"有多少迁移"）更稳。

4. **圈层归属的粒度？** 目前队友用该小时最后一条记录的圈层作为该船的圈层（`zone.py:73`）。如果船在一小时内跨圈，这个策略可能偏。

5. **提交模板在哪？** 目前 `train.py` 依赖 `data/raw/提交结果1_区域活跃拖轮数量.csv` 等模板文件，尚未生成。需要根据预测时段构造。
